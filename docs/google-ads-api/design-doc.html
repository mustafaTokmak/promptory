Design Document

Promptory — Google Ads API Integration

Product: Promptory — AI Prompt Directory for ChatGPT (Chrome Web Store extension) · MCC: 470-629-5146 · Contact: info@promptory.chat · Site: promptory.chat

1. Overview

Promptory is a free Chrome extension distributed through the Chrome Web Store. We use Google Ads (Performance Max + Search) to drive installs. Because the Chrome Web Store is a third-party domain, we cannot place a Google Ads conversion tag on the install page — so we use the Google Ads API ConversionUploadService.UploadClickConversions endpoint to upload offline install conversions from our backend, attributed via the gclid we capture on a bridge page on our own domain.

2. Architecture

[Google Ad click] │ ▼ [promptory.chat/go?gclid=…] stores gclid in localStorage, redirects │ ▼ [chromewebstore.google.com/detail/<ext-id>] user installs extension │ ▼ [chrome.runtime.onInstalled → opens promptory.chat/setting-up] │ ▼ [/setting-up reads gclid, POSTs to api.promptory.chat/v1/conversion] │ ▼ [Promptory backend (Cloudflare Workers) → Google Ads API] │ ▼ [ConversionUploadService.UploadClickConversions → conversion attributed]

3. API Usage

One endpoint, called server-side only:

POST googleads.googleapis.com/v17/customers/4706295146:uploadClickConversions
{
  "conversions": [{
    "gclid": "Cj0K…",
    "conversionAction": "customers/4706295146/conversionActions/<id>",
    "conversionDateTime": "2026-05-02 15:42:01+03:00",
    "conversionValue": 0.0,
    "currencyCode": "USD"
  }],
  "partialFailure": true
}

Expected volume: under 5,000 calls/day in the first 12 months. Other read-only calls (ListAccessibleCustomers, GetConversionAction) used at deploy time only.

4. Authentication & Security

  • OAuth 2.0 with one Google account owning the MCC; refresh token + developer token stored as Cloudflare Workers secrets.
  • Access token cached in Workers KV until expiry, refreshed on demand.
  • POST /v1/conversion rate-limited (60 req/min/IP), CORS-restricted to promptory.chat, gclid format-validated before upload.
  • HTTPS everywhere. Raw gclids never logged; redacted hashes only.

5. Access Scope

Internal only. Promptory's backend is the sole caller. End users never authenticate against Google Ads, never see API responses, and never trigger calls beyond the implicit conversion upload that happens server-side after install. No third-party access. Applying for Basic Access.

6. Privacy & Retention

  • No PII collected. gclid in localStorage is removed after successful upload (max 30 days).
  • Backend stores gclid + timestamp in Cloudflare D1 for 90 days for audit, then deletes.
  • Privacy policy: https://promptory.chat/privacy.
